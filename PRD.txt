Project PRD

Project Name: TeleDrive (working title)

Goal:
A lightweight browser-based cloud storage service using Telegram Bot API as storage and Cloudflare Workers as the backend layer.

The system should support authenticated personal storage, sharing links, file management, and preview capabilities while keeping server infrastructure minimal and mostly client-side.

Core Architecture
Storage Layer

Uses the Telegram Bot API.

Entity: telegram_file

Each uploaded chunk is stored as a Telegram message in a private channel.

Stored metadata:

file_id
message_id
channel_id
chunk_index
chunk_size
checksum

Files larger than 50MB are split client-side.

Backend Layer

Runs on Cloudflare Workers

Responsibilities:

• authentication
• metadata management
• file indexing
• share link generation
• permission checks
• message retrieval from Telegram

Workers never process file data, only metadata.

Database

Use Cloudflare D1 or Cloudflare KV

Recommended: D1 (SQL) because folders and ownership relationships are easier.

Client Responsibilities

All heavy work happens in the browser.

Client tasks:

File Upload

user selects file

browser splits file into chunks

chunks uploaded sequentially to worker

worker forwards to Telegram

Chunk size recommendation:

8MB

Reason:
Safe margin below Telegram limit.

After upload:

client sends chunk metadata to worker
worker records chunk order
File Download

client requests file metadata

worker returns chunk list

client downloads chunks

browser merges chunks

file delivered to user

Chunk merging uses:

Blob()
Streaming Preview
